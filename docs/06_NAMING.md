# MeshDraw — Naming
> Working codename: **MeshDraw**. Decide before public-facing UI/marketing.

## Why "MeshDraw" undersells us
It names the *input* (draw) and *output* (mesh) — the two commodity parts. It says nothing about the actual innovation (intent / living, parametric assets). Sounds like a *feature*, not a *platform*.

## Decision framework — pick the FEELING first
The name should carry ONE of these:
- **(A) Draw** — the input act. (MeshDraw, Sketchbook3D...)
- **(B) Living / malleable** — clay never dries, geometry never "dies." (Clay, Morph, Weave, Volu)
- **(C) AI collaborator that preserves authorship** — (Muse, Copilot-for-3D vibe)

> Recommendation: lean **(B) living/malleable** — it's the true differentiator and hardest to copy.

## Candidates
| Name | Feeling | Pros | Cons |
|------|---------|------|------|
| **Clay** | B | perfect metaphor (always reshapeable, never dead) | common word, TM/SEO fight |
| **Morph** | B | strong verb; UI + mesh both morph | possibly crowded |
| **Weave** | B | strokes weave into form; elegant | abstract |
| **Volu / Voluma** | B | volume+living; ownable/brandable | invented, needs meaning-building |
| **Muse** | C | "collaborator, not replacer" | common |
| **Loft** | A/B | insider 3D verb (loft/revolve/extrude); ownable | niche/jargon |
| **Ideate / Idea Engine** | thesis | on-theme ("object is an idea") | generic, used |
| **MeshDraw** | A | clear, functional | undersells; feature-y |
